# 什麼是 Vibe Coding？入門概念全解析

**作者：** Danko Peng（[@dankopeng](https://x.com/dankopeng)）

**閱讀時間：** 約 8 分鐘

## 先從一個真實場景開始

想像你腦中有一個 app 的想法。

以前，把這個想法變成真實產品，你需要：學會一門程式語言、搞懂前端後端的區別、設定資料庫、處理部署環境、解決一堆看不懂的錯誤訊息……光是準備工作就要好幾個月。

現在不一樣了。

你可以用**自然語言**告訴 AI 你想做什麼，AI 幫你寫 code，你負責判斷、測試、調整方向。整個流程更像是「和一個會寫 code 的助理合作」，而不是自己從零學一門技術。

這就是 **Vibe Coding**。

-----

## Vibe Coding 的定義

**Vibe Coding** 是一種以 AI 為主要開發工具、人負責引導方向和判斷結果的軟體開發方式。

這個詞由 OpenAI 共同創辦人 **Andrej Karpathy** 在 2025 年初提出。他描述自己完全沉浸在 AI 輔助開發的狀態裡，幾乎不自己寫 code，只負責給 AI 指令、測試結果、調整方向——整個過程有一種「跟著感覺走」的流動感，所以叫 Vibe Coding。

核心概念很簡單：**你描述你想要什麼，AI 負責生成 code，你負責判斷對不對。**

-----

## 和傳統寫 code 有什麼不同？

|     |傳統開發       |Vibe Coding         |
|-----|-----------|--------------------|
|主要工作 |自己寫每一行 code|用自然語言描述需求，AI 生成 code|
|需要的知識|深入掌握程式語言   |理解基本概念，能判斷 AI 輸出    |
|速度   |較慢，學習曲線陡   |快，從想法到原型可以在幾小時內完成   |
|適合人群 |有紮實技術基礎的工程師|任何有想法、願意學習判斷 AI 輸出的人|
|風險   |錯誤由自己承擔    |AI 可能生成錯誤 code，需要會識別|

Vibe Coding 不是說完全不需要懂技術——你還是需要理解基本概念，才能判斷 AI 說的對不對。但進入門檻確實低很多，而且速度快很多。

-----

## 誰適合 Vibe Coding？

**非常適合：**

- 有產品想法但沒有工程背景的創業者
- 想快速驗證想法的 solopreneur
- 設計師、產品經理想自己動手做 prototype
- 有一點基礎但卡在「做不出完整產品」的開發者
- 像我這樣 50 歲才開始認真 build 產品的人

**不太適合：**

- 需要極高可靠性的系統（金融、醫療核心系統）
- 對安全性要求極嚴格的基礎設施

對大多數想快速上線 web app、驗證商業想法的獨立開發者來說，Vibe Coding 是目前最有效率的方式。

## 核心術語一次搞懂

進入 Vibe Coding 的世界，你會遇到一堆術語。這裡把最常見的幾個說清楚：

-----

### 前端 / 後端 / 全棧

**前端（Frontend）** 是用戶看得到、點得到的部分——網頁的樣子、按鈕、表單。  
**後端（Backend）** 是用戶看不到的部分——處理登入邏輯、存資料、計算結果。  
**全棧（Fullstack）** 是前端加後端都包含。

VibeFast 是一個全棧模板，前端用 **Remix**，後端用 **Cloudflare Workers**。

-----

### Edge 邊緣運算

傳統的伺服器放在某個固定地點（例如美國某個機房）。當台灣的用戶訪問你的網站，資料要跑到美國再跑回來，有延遲。

**Edge（邊緣）** 的概念是：把你的 code 和資料分散部署到全球各地的節點，用戶的請求自動導向最近的節點處理。台灣的用戶就近處理，美國的用戶也就近處理，速度都很快。

Cloudflare 有 300+ 個邊緣節點，VibeFast 部署上去之後，全球用戶都能快速訪問。

-----

### Cloudflare Workers

**Workers** 是 Cloudflare 提供的後端執行環境。你的後端 code 跑在 Workers 上，不需要自己管伺服器。

傳統後端要租一台虛擬機、安裝環境、設定防火牆……Workers 讓你只需要寫 code，其他都 Cloudflare 幫你處理。

更重要的是，Workers 跑在 **邊緣**，不是某個固定機房，所以速度快、沒有冷啟動問題。

-----

### Cloudflare D1

**D1** 是 Cloudflare 提供的資料庫服務。技術上是 SQLite（一種輕量資料庫），但跑在 Cloudflare 的邊緣網路上。

你的 app 需要儲存用戶資料、文章內容、訂單記錄——這些都放在 D1 裡。D1 就在 Workers 旁邊，讀寫速度非常快。

對 Vibe Coder 來說，D1 的好處是：可以直接告訴 AI「幫我設計一個儲存用戶資料的表格」，AI 生成的 SQL 直接能用。

-----

### Cloudflare R2

**R2** 是 Cloudflare 提供的檔案儲存服務，功能類似 Amazon S3，用來放圖片、影片、PDF 等媒體檔案。

R2 最大的優勢是**零出站費用**——傳統雲端儲存（如 AWS S3）在用戶下載檔案時會收取出站流量費，R2 不收。

-----

### Service Binding

這是 Cloudflare 特有的概念。

在 VibeFast 裡，前端（Remix）和後端（Workers API）是兩個獨立的服務。傳統做法，前端要通過公開網路呼叫後端 API，這會有 CORS 問題（跨域請求的安全限制），需要額外設定。

**Service Binding** 讓前端和後端在 Cloudflare 內部直接溝通，不走公開網路。結果是：零 CORS 設定、速度更快、API 不暴露在外面。

對 Vibe Coder 來說，這意味著少了一個需要設定和踩坑的環節。

-----

### Monorepo

**Monorepo** 是把多個相關的 code 放在同一個 GitHub 倉庫（repository）裡管理。

VibeFast 的 monorepo 結構：前端 code、後端 code、共用元件，全部在同一個 repo 裡，用 **Turborepo** 統一管理建構流程。好處是不需要在多個 repo 之間切換，共用的 code 也容易維護。

-----

### Boilerplate

**Boilerplate** 指的是每個新專案都需要重複設定的基礎 code——例如登入系統、資料庫連線、部署設定。

這些 code 不是你的產品核心，但每次都要重寫，很花時間。VibeFast 這類模板就是幫你把 boilerplate 都準備好，讓你直接從「產品功能」開始。

-----

### JWT（JSON Web Token）

**JWT** 是一種用戶身份驗證的技術。

用戶登入之後，系統給他一個加密的「通行證」（token），之後每次請求都帶著這個通行證，系統驗證後知道「這個用戶是誰、有什麼權限」。

VibeFast 內建 JWT auth，不需要自己實作。

-----

### Migration（資料庫遷移）

當你的 app 成長，資料庫的結構需要改變——例如新增一個欄位、改一個表格名稱。

**Migration** 是管理這些變更的方式，讓你可以追蹤資料庫結構的歷史，也可以在不同環境（開發環境、正式環境）之間同步。

-----

## 總結：Vibe Coding 的三個核心心態

學完這些概念，我想用三句話來總結 Vibe Coding 的精髓：

**1. 少想、多試**  
不要等到完全搞懂才開始。先讓 AI 生成一個版本，跑起來看看，再調整。

**2. 你是導演，AI 是執行**  
AI 負責寫 code，你負責判斷方向對不對、結果符不符合預期。這個角色分工很重要。

**3. 選好平台，省掉大半麻煩**  
工具和平台選對了，很多技術問題會自動消失。Cloudflare 全棧就是為了讓你少踩坑而設計的。
